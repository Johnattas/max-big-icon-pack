## 💡 O que é

O **Max Big Icon Pack** é um **File Icon Theme** para o Visual Studio Code que parte da elegância flat do consagrado **[Material Icon Theme](https://github.com/material-extensions/vscode-material-icon-theme)** e o expande com uma curadoria de **mais de 10.000 ícones** de linguagens, frameworks, ferramentas, serviços e marcas — todos no mesmo estilo visual coerente.

O objetivo é simples: **qualquer arquivo ou pasta que você abrir deve ter um ícone que faça sentido.** Do `.pinia` ao `dockerfile`, da pasta `stripe/` à `worktrees/`, o reconhecimento visual é imediato.

> Todos os ícones são **gerados por um pipeline reproduzível** a partir de bibliotecas open-source (Material, file-icons/Atom, A File Icon, Simple Icons e Devicon). Nada de arte inventada nem inconsistente — só ícones que combinam entre si.

## ✨ Destaques

- 💎 **Biblioteca gigante (10.240+ ícones)** — cobre praticamente qualquer linguagem, framework, banco de dados, ferramenta de DevOps, serviço ou marca que você use.
- 🎨 **Estilo 100% coerente** — tudo é flat, no espírito do Material. Marcas viram ícones de pasta com o logo **composto sobre a base de pasta** (fechada e aberta), não logos soltos.
- 🧠 **Reconhecimento inteligente de pastas** — `storage`, `http`, `ai`, `traits`, `observers`, `workflows`, `webhooks`, `sessions`, `requests`, `leads`, `.superpowers`, `.adonisjs`, `mcp` e dezenas de outras já vêm mapeadas.
- 🚦 **Manifestos vs. locks coloridos** — `package.json` 🟢 verde e `package-lock.json` 🔴 vermelho; idem `composer.json`/`composer.lock`. Distinção instantânea no explorer.
- 🔁 **Reproduzível e auditável** — um comando (`npm run import-icons`) regenera o tema inteiro a partir das fontes. Nada é editado à mão no JSON final.
- 🌐 **Multi-editor** — funciona no VS Code, Cursor, Windsurf, VSCodium, Gitpod e qualquer editor compatível com o formato `.vsix`.

## 🛠️ Instalação

### Via arquivo `.vsix`

O pacote instalável fica em [`dist-theme/max-big-icon-pack-1.0.22.vsix`](dist-theme/):

```bash
code --install-extension dist-theme/max-big-icon-pack-1.0.22.vsix
```

> Compatível também com **Cursor**, **Windsurf**, **VSCodium** e outros editores que aceitam o formato VSIX.

### Gerando do zero (opcional)

```bash
npm install
npm run import-icons          # gera dist-theme/ (tema + ícones)
cd dist-theme && npx @vscode/vsce package   # empacota o .vsix
```

## ▶️ Como ativar

1. Abra a Paleta de Comandos:
   - **Windows/Linux:** `Ctrl+Shift+P`
   - **macOS:** `Cmd+Shift+P`
2. Digite **`Preferences: File Icon Theme`**.
3. Selecione **`Max Big Icon Pack`**.

## 📚 De onde vêm os ícones

Diferente do que muitos "mega packs" fazem, aqui **não** misturamos temas de desktop (KDE/GTK) que destoariam do estilo. Cada ícone vem de uma biblioteca open-source flat, na ordem de prioridade:

| Fonte | O que contribui |
|---|---|
| **[Material Icon Theme](https://github.com/material-extensions/vscode-material-icon-theme)** | Base completa (arquivos + pastas) — o alicerce visual |
| **[file-icons/atom](https://github.com/file-icons/atom)** | Glifos de fonte extraídos e colorizados como SVG |
| **[A File Icon](https://github.com/SublimeText/AFileIcon)** | Ícones de tipo de arquivo (PNG) para lacunas |
| **[Simple Icons](https://simpleicons.org/)** | ~3.000 logos de marcas/tecnologias (colorizados) |
| **[Devicon](https://devicon.dev/)** | Linguagens e ferramentas de desenvolvimento |

Ícones que faltam no Material são detectados por **diff automático** contra a cobertura atual; duplicatas (ex.: `nextjs` → `next`) são removidas na verificação.

## 🏗️ Como é construído (pipeline)

Todo o tema é **gerado**, não montado à mão. O código vive em [`scripts/import-icons/`](scripts/import-icons/) e roda com:

```bash
npm run import-icons
```

O que ele faz, em etapas isoladas e testadas:

1. **Base** — carrega o manifest completo do `material-icon-theme` e copia seus SVGs.
2. **Lacunas** — calcula, por diff, os conceitos/associações que faltam.
3. **Geração** — extrai glifos do Atom (via `fonttools`), copia PNGs do AFileIcon, colore logos do Simple Icons/Devicon.
4. **Composição de marca** — compõe cada logo sobre a base de pasta Material (estado fechado **e** aberto).
5. **Overrides** — recolore pastas, mapeia nomes de pasta e aplica o código de cor de manifestos/locks.
6. **Validação** — confere que todo `iconDefinition` aponta para um arquivo existente e que nenhuma associação fica órfã.

A saída (`dist-theme/max-big-icon-theme.json` + `dist-theme/icons/`) é empacotada como extensão VSIX. Cobertura de testes com **Vitest** e lint com **Biome**.

## 🎛️ Personalizando

Este é um icon theme **estático** — a personalização é feita editando as tabelas do pipeline e regenerando (não pelas configs `material-icon-theme.*`, que pertencem à extensão original).

**Mapear uma pasta para um ícone** — em [`scripts/import-icons/lib/custom-overrides.ts`](scripts/import-icons/lib/custom-overrides.ts), na tabela `FOLDER_ICONS`:

```ts
export const FOLDER_ICONS: Record<string, string> = {
  // ...
  minha_pasta: 'folder-api',
};
```

**Recolorir um arquivo (manifesto/lock, etc.)** — na tabela `FILE_RECOLORS`:

```ts
export const FILE_RECOLORS: FileRecolor[] = [
  // ...
  { fileName: 'yarn.lock', from: 'yarn', color: '#e53935' },
];
```

Depois é só rodar `npm run import-icons` e reempacotar. Simples e versionável.
